# 成交量异动策略（Volume Anomaly）AI 量化优化方案

基于对当前 `volume_anomaly` 插件代码库（特别是 `data.py` 和 `prompts.py` 等核心逻辑）的深入研究，现有的策略已经建立了一个非常扎实的框架：通过本地硬性条件（量价齐升、均线、基础形态）进行漏斗筛选，再由 LLM 进行多维度的定性评估（洗盘质量、形态结构、资金及板块验证）。

为了进一步提升该策略“捕获主升浪”的胜率和收益，现从 AI 量化的角度提出以下优化建议，主要涵盖筛选条件、因子扩充以及 LLM 推理层面的演进。

---

## 1. 筛选条件优化 (Screening Rules)

当前的本地筛选规则 (`ScreenRules`) 采取了较为严格的绝对数值阈值。在量化实战中，固定的绝对数值容易在不同的市场环境或不同市值的标的中失效。

*   **动态换手率阈值 (Dynamic Turnover Rate)**
    *   **现状**：硬编码换手率区间 `[3.0, 10.0]`。这忽略了盘子大小的区别（例如千亿市值的大盘股 5% 已经是天量，而 20 亿市值的小微盘 10% 属于正常活跃）。
    *   **优化方案**：引入基于流通市值的动态换手率区间，或使用**相对换手率分位数**。例如，要求当天的换手率大于其自身过去 60 个交易日换手率的 80% 分位数（`turnover_t > quantile(turnover_60d, 0.8)`）。
*   **形态过滤精细化 (Candlestick Shape Control)**
    *   **现状**：涨幅限制在 `[5.0, 8.0]`，实体比例 `body_ratio >= 0.6`。
    *   **优化方案**：虽然 `body_ratio` 控制了阳线实体，但未能严格限制上影线的长度。放量异动时，如果上影线过长（“避雷针”形态），往往是主力试盘失败或高位派发。建议增加 `upper_shadow_ratio < 0.3`（上影线占全天振幅比例）的过滤条件。同时，将绝对涨跌幅稍微放宽（如 `[4.0, 9.5]`），避免漏掉形态极好但收盘略微回落的标的。
*   **多日量价堆积识别 (Volume Accumulation)**
    *   **现状**：当前要求单日放量（`vol_t >= 2.0 * mean(prev 5d)`）。
    *   **优化方案**：单日暴量有时是陷阱，而连续 3-5 天的“温和放量”（成交量阶梯式递增，如 $V_{T} > V_{T-1} > V_{T-2}$ 且伴随价格抬升）是更坚实的主升浪前兆。建议在筛选层允许这类“量价齐升堆积”模式作为 `vol_t >= 2.0` 的并列通过条件。

## 2. 补充量化因子 (Supplementary Factors)

目前输入给 LLM 的数据主要包含 60 日均线、基础的洗盘指标（区间最大回撤、缩量比例）以及 5 日资金流。为了让 LLM 做出更准确的判断，建议补充以下经典量化技术因子：

*   **波动率收缩特征 (VCP, Volatility Contraction Pattern)**
    *   **原理**：Mark Minervini 的 VCP 理论是捕捉主升浪的经典模型。爆发前往往伴随着价格波动的极度收敛。
    *   **新增因子**：加入 60 日的**布林带宽度 (Bollinger Band Width)**，或者计算近期 10 天的真实波动幅度 (ATR) 在过去 60 天的分位数。如果一只股票在异动放量前，其 ATR 处于历史极低位（如 10% 分位），说明洗盘极其彻底，这对于 LLM 评估 `washout_quality` 具有极高价值。
*   **相对强弱指标 (RPS, Relative Price Strength)**
    *   **原理**：牛股在启动前往往比大盘更抗跌。
    *   **新增因子**：提供标的相较于万得全 A 或上证指数的 20 日/60 日相对收益（Alpha）。如果某标的在近期大盘连续下跌时保持横盘或微涨（RPS > 80），那么它在市场企稳时极易率先启动。
*   **关键阻力位距离 (Distance to Resistance)**
    *   **原理**：突破前高的放量和在下跌通道中反弹的放量，其技术意义完全不同。
    *   **新增因子**：计算当前收盘价距离过去 120 日或 250 日最高点的距离百分比。这能帮助 LLM 更好地判断当前的 `pattern` 到底是真正的 `breakout`，还是被套牢盘压制的超跌反弹。

## 3. LLM 提示词与推理优化 (Prompt & Reasoning)

在当前的 `VATrendCandidate` 结构中，LLM 已经具备了结构化输出的能力，但还可以进一步引导其推理过程。

*   **引入假突破风险评估 (Bull Trap Risk)**
    *   **优化**：在判断维度中，强制要求 LLM 评估“诱多出货”的概率。如果存在长上影线、大单净流出 (`net_mf_trend` == falling) 与放量阳线背离、或者处于板块炒作末期等现象，强制 LLM 在 `risk_flags` 中输出类似 `High Bull Trap Risk` 的标签，并在 `rationale` 中予以否定（给出 `not_yet`）。
*   **结构化思维链 (Structured Chain-of-Thought)**
    *   **现状**：目前的 `rationale` 是单一的 200 字字符串，模型可能在不同的判定维度间跳跃。
    *   **优化**：可以将其拆分为几个内部小结（哪怕只是在思维过程中），强制模型先打分后汇总。例如，在提示词中建议 LLM 遵循：`(1) 洗盘深度评估 -> (2) 形态突破有效性 -> (3) 资金与大势共振 -> (4) 结论` 的逻辑链进行思考，这样可以大幅减少“看到放量就盲目看多”的幻觉。
*   **交易视角的盈亏比预估 (Risk/Reward Estimation)**
    *   **优化**：作为“助手”，可以要求 LLM 在给出 `imminent_launch` 判断时，同时基于输入的近期低点或均线，提出逻辑上的“防守位/止损位”（例如：`next_session_watch: ["如果跌破 MA10 且无法收回，则主升浪逻辑破坏"]`）。这种视角的转变能促使 LLM 以真实交易员的心态去审视启动的安全边际。
*   **Few-Shot 示例对齐标准**
    *   **现状**：Prompt 中只有 Schema 规范，缺乏判断尺度的基准。
    *   **优化**：在 `VA_TREND_SYSTEM` 的提示词中，增加 1-2 个极简的 JSON 示例。一个展示“教科书式的 VCP 洗盘后放量突破”（给出 `imminent_launch` 和 `sufficient` 的范例），另一个展示“高位巨量烂板，资金流出”（给出 `not_yet` 和 `insufficient` 的范例）。这能最直接地对齐不同大语言模型的评判标准，降低因为模型更换（如从 GPT-4 换为 Claude-3）带来的表现波动。

## 4. 全局 AI 量化架构与交易系统演进 (System-Level & Advanced AI Quant Architecture)

除了上述“特征工程”和“提示词优化”这类较为直接的改进外，将视野扩展到更深度的 AI 量化架构和交易系统全局，本策略还有很大的前沿演进空间：

*   **引入更细粒度的数据维度 (Intraday & Alternative Data)**
    *   **日内分时特征 (Intraday Volume Profile)**：同样是日线放量 2 倍，“早盘 30 分钟爆量封板”与“全天烂板震荡放量”或“尾盘 10 分钟偷袭放量”背后的主力意图截然不同。建议在异动当天（T日），提取日内核心特征（例如：前一小时成交量占比、尾盘一小时资金净流、日内最大回撤等）喂给 LLM，比单纯看日线极值更能识别“真洗盘”与“假突破”。
    *   **板块与题材的动态图谱 (Knowledge Graph 共振)**：目前输入了板块强度摘要，但较为静态。可通过 AI 构建微型的题材映射关系：如果该标的发生异动，且其所属概念正好是当日或近三日市场资金净流入前三的绝对主线，直接给予“主线共振加分权重”，强化宏观与中观的结合。
*   **架构演进：从单节点 LLM 到“大小模型协同 / 多智能体” (Multi-Agent & Hybrid Models)**
    *   **传统 ML 与 LLM 结合 (LightGBM/XGBoost + LLM)**：对于“60日涨幅、波动率、资金流”等数值特征，先用基于历史数据训练的 LightGBM 树模型，计算出一个“3日内爆发概率预测分 (Propensity Score)”。再将分数与数据一并交给 LLM。LLM 的角色转变为“对高分标的进行形态和逻辑的最终把关与否决（Veto）”，这能极大提高系统的准确率和逻辑自洽性。
    *   **多专家智能体 (Multi-Agent Debate)**：对于异动股，拆分两个 Prompt 角色进行博弈：
        *   **红方（做多验证者）**：寻找为什么这是主升浪起点的证据（如缩量极好、大单坚决）。
        *   **蓝方（风险排雷者）**：专门挑刺，寻找诱多特征（如左侧套牢盘重、大盘环境恶劣）。
        由“决策者 Agent”综合双方观点给出最终 `launch_score`，大幅降低 LLM 的幻觉和单向偏见。
*   **构建闭环的自我进化系统 (Self-Reflection & Feedback Loop)**
    *   **归因与动态经验库 (Post-Mortem RAG)**：对于预测为即刻启动的标的，系统在 3-5 个交易日后自动回测（T+3 收益率）。若预测失败，触发“复盘 Agent”分析当时的输入数据和错误逻辑，生成教训（Lesson）存入向量数据库。下次预测时，检索相似场景的历史教训作为动态 Prompt 注入。使策略能像人类交易员一样积累盘感、越挫越勇。
*   **交易执行层的数值化输出 (Actionable Execution Metrics)**
    *   **动态止损止盈位计算 (Dynamic Risk Levels)**：要求 LLM 根据历史支撑位、缺口和近期大阳线的开盘价，直接输出具体的防守价格数值（如 `stop_loss_price: 15.30`，`target_price: 18.50`）。这允许下游实盘或回测模块直接挂单，并自动计算预期盈亏比 (Risk-Reward Ratio)，从而直接指导仓位管理（如应用凯利公式）。

---

通过上述优化，策略将从一个“严谨的量化指标读取器”逐步进化为一个具备**“量化前置打分、日内微观视角补充、多方博弈思维以及交易后自我反思”**能力的智能化虚拟基金经理系统。
